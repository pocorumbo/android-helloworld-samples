Handler to communicate between threads

In Android applications is not uncommon to create threads that perform time consuming tasks. By doing so we avoid latency in the main UI thread. But how do we communicate back to the UI thread from these other threads? The Handler comes in handy in such situations. A Handler allows you to send and process Message and Runnable objects associated with a thread's MessageQueue. Each Handler instance is associated with a single thread and that thread's message queue.

When we create a new Handler, it is bound to the thread / message queue of the thread that is creating it. We can pass this Handler object to other threads so that they can post back messages to the messagequeue of the thread in which this handler was originally created.In this sample application we are using a Handler object to communicate from a user defined thread back to the main UI thread.
